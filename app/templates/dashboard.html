Project Overview

AI-assisted classification of ferrofluid research papers and patents using GPT-4o + Claude dual consensus.

About This Project

This system classifies ~4,500 ferrofluid / magnetic fluid research documents (papers and US patents) into 30 predefined class codes across 5 major categories: Material, Computation, Experimentation, Application, and Review/Book.

Each document is classified by two independent AI models (OpenAI GPT-4o and Anthropic Claude Sonnet) based solely on its abstract text. Each model assigns a Primary (most relevant), Secondary, and Tertiary class code along with reasoning.

Understanding the Stats

  • Agreed — Both AI models chose the same primary class code. The classification is automatically accepted.
  • Disagreed — The models chose different primary codes. These need human review to decide which is correct.
  • Human Reviewed — A human has manually resolved the disagreement and selected the final classification.
  • Pending — Documents that have not yet been classified (no abstract, or not yet processed).
  • Classified — Total documents that have been through the AI pipeline (agreed + disagreed + human reviewed).

Goal 1: Classified Papers

What is Goal 1?

Classify every research paper from the Scopus dataset into the 30-code ferrofluid taxonomy. The output is a sorted CSV preserving all original columns from the source data, with added classification columns.

How to Read This Table

  • Serial — Unique identifier (P1, P2, ...). "P" = paper.
  • Year — Publication year.
  • Title — Paper title (hover to see full text).
  • Primary Class — The most relevant class code assigned to this paper. This is the main classification.
  • Secondary Class — The second most relevant class code.
  • Tertiary Class — The third most relevant class code. If a paper covers only one clear subject, all three may be the same.
  • Status — "agreed" means both AI models agreed on the primary code; "human_reviewed" means a human resolved a disagreement.

Color-coded badges show the major category: green = Material, blue = Computation, orange = Experimentation, pink = Application, purple = Review/Book.

Sorted by: Year → Primary Class → Secondary Class → Tertiary Class. Only finalized documents (agreed or human-reviewed) appear here. Unresolved disagreements are excluded until reviewed.

Goal 2: Classified Patents

What is Goal 2?

Classify every US patent from the WIPO/Derwent datasets (1969–present) into the same 30-code ferrofluid taxonomy used for papers. Two patent CSV files are combined: patents from 1969–2009 and 2010–present.

How to Read This Table

  • Serial — Unique identifier (PT1, PT2, ...). "PT" = patent.
  • Year — Publication year of the patent.
  • Title — Patent title.
  • Primary / Secondary / Tertiary Class — Same classification system as papers. The AI reads only the patent abstract to assign codes.
  • Status — Same as papers: "agreed" or "human_reviewed".

This table uses the same format, sort order, and badge colors as the papers table. Comparing papers and patents by their class codes is how we identify research-to-patent gaps (Goal 3a).

Goal 3a: Gap Analysis

What is Gap Analysis?

Gap analysis answers the question: "Where does research exist but patents don't, and vice versa?" For each of the 30 class codes, we count how many papers and patents were classified into that class, then identify the gaps.

Charts

  • Papers vs Patents by Class (bar chart) — Each bar pair shows how many papers (blue) and patents (purple) fall into a given class code. Tall blue bars with no purple = research with no patent activity.
  • 5-Year Periods (line chart) — Tracks the top 6 most active class codes over time in 5-year buckets, showing how research and patent activity has evolved over decades.

Gap Type Legend

  • both — The class has at least one paper AND at least one patent. Active in both research and industry.
  • papers only — Research exists but no patents have been filed. This is a commercialization gap — potential opportunity for new patents.
  • patents only — Patents exist but no research papers. Rare; suggests industry activity without published academic research.
  • empty — No documents of either type in this class.

Goal 3b: Patent-Paper Links

What is Patent-Paper Linking?

For each classified patent, we find the 3 most related research papers based on the textual similarity of their abstracts. This creates a relational dependency between the patent and paper datasets.

How It Works

We use TF-IDF (Term Frequency–Inverse Document Frequency) to convert every abstract into a numeric vector, then compute cosine similarity between each patent and all papers. The top 3 papers with the highest similarity score are linked to each patent.

How to Read This Table

  • Patent / Paper — The serial numbers and titles of the linked documents.
  • Similarity — A score from 0% to 100%. Higher = more similar abstracts. A score above 20% generally indicates meaningful topical overlap. Scores above 40% indicate strong similarity.

Each patent appears up to 3 times (once per linked paper), sorted by similarity score descending.

Goal 4: Assignee Cross-Reference

What is Assignee Cross-Referencing?

This goal identifies individuals who appear as both patent inventors/assignees AND research paper authors working on the same ferrofluid topic (same primary class code). These people bridge academia and industry.

How It Works

We extract inventor names from patents (including Applicants and Owners columns) and author names from papers. Names are normalized (lowercase, "Last, First" → "first last") and matched. A match is recorded only if the person worked on the same primary class in both their patent and paper.

Why 0 Matches?

The patent dataset uses inventor names in a different format (often abbreviated or organization names) than the paper dataset (full author names with initials). This formatting mismatch prevents exact string matching from finding overlaps. Fuzzy name matching or manual curation would be needed to improve results.

Knowledge Graph

What is the Knowledge Graph?

An interactive network visualization that shows how the 30 ferrofluid class codes relate to each other across the entire dataset.

How to Read It

  • Square nodes — Each square represents a class code. Its size reflects how many documents are in that class (bigger = more documents).
  • Node colors — Colored by major category: green = Material, blue = Computation, orange = Experimentation, pink = Application, purple = Review/Book.
  • Red nodes — These are GAP classes: they have research papers but NO patents. These represent potential commercialization opportunities.
  • Gray edges — Lines connecting two class codes indicate co-occurrence — documents that have one class as primary and another as secondary. Thicker edges = more co-occurring documents.

You can drag nodes, zoom in/out, and hover over any node to see detailed stats (paper count, patent count, description). Classes that cluster together share common topics.
